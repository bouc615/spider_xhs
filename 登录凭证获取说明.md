# 登录凭证获取说明

## 📋 简化方案

经过用户反馈，移动端获取登录凭证的方法过于复杂，现在采用简化方案：

### 💻 推荐方法：使用电脑

1. 打开小红书网页版 (xiaohongshu.com) 并登录
2. 按 F12 打开开发者工具
3. 切换到 Network(网络)标签
4. 刷新页面，找到任意请求
5. 点击请求，在 Request Headers 中找到 Cookie 字段
6. 复制完整的 Cookie 值（这就是你的登录凭证）

### 📱 移动端用户

建议找一台电脑获取登录凭证，然后通过 QQ、微信等发送给自己，在手机上复制粘贴即可。

## ✅ 登录凭证管理功能

项目已实现完整的登录凭证管理系统：

- **格式验证**: 自动检查登录凭证格式和必要字段
- **有效性测试**: 调用小红书 API 验证登录凭证是否有效
- **本地存储**: 登录凭证保存在浏览器 localStorage 中
- **状态显示**: 实时显示登录凭证配置状态
- **用户信息**: 测试成功时显示当前登录用户名

## 💡 为什么不支持自动登录？

1. **技术复杂性**: 小红书登录涉及复杂加密和验证
2. **反爬虫对抗**: 自动登录容易被检测封禁
3. **稳定性考虑**: 手动登录凭证方式更稳定可靠
4. **安全性**: 避免存储敏感的登录密码

## ⚠️ 注意事项

- 登录凭证通常有效期为 7-30 天
- 失效后需要重新获取
- 不要泄露登录凭证给他人
- 定期测试登录凭证有效性

## 🔍 什么是登录凭证？

登录凭证实际上就是浏览器中的 Cookie 信息，它包含了你的登录状态和身份验证信息。当你在浏览器中登录小红书后，这些信息会自动保存，我们只需要复制这些信息就可以让程序代替你访问小红书的数据。

这就像是你的"数字身份证"，证明你已经登录过小红书，有权限访问相关数据。
